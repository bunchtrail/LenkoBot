# Governance

LenkoBot currently uses a maintainer-led governance model appropriate for an
early public project.

## Maintainer

[Nikita Lolenko (@bunchtrail)](https://github.com/bunchtrail) is the founding
maintainer and current repository owner. The maintainer is responsible for
release decisions, security response, repository access, architecture contracts,
and final review of contributions.

## Decisions

Routine fixes are decided in pull-request review. Changes to architecture,
persisted data, public interfaces, security policy, or user workflows begin with
a documented issue or proposal. Decisions are based on repository evidence,
tests, operational cost, and the project's single-owner trust model.

Security-sensitive decisions may be discussed privately until a coordinated fix
is available. Accepted long-lived decisions are recorded in the architecture
documents rather than left only in issue comments.

## Becoming a maintainer

Maintainer access may be offered after a sustained record of reviewed
contributions, sound security judgment, reliable follow-through, and alignment
with the documented project boundaries. There is no contribution-count shortcut.

As the maintainer group grows, this document will be updated before permissions
or release authority are delegated.
